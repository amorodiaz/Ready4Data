📊 Fundamentos de datos
Base de datos
Colección organizada de información almacenada en tablas. Piensa en un Excel muy bien ordenado y potente. Ejemplo: la base de datos de clientes de Amazon.
Tabla
Estructura con filas y columnas que almacena un tipo de información. Como una hoja de Excel. Ejemplo: tabla de pedidos con columnas id, fecha, importe, cliente_id.
Fila / Registro
Una entrada individual en una tabla. Si la tabla es "clientes", cada fila es un cliente concreto.
Columna / Campo
Un atributo de la tabla. En la tabla "clientes" las columnas serían: nombre, email, ciudad, fecha_registro.
Clave primaria (PK)
Columna que identifica de forma única cada fila. Normalmente llamada id. No puede repetirse ni estar vacía.
Clave foránea (FK)
Columna que apunta a la clave primaria de otra tabla. Crea relaciones entre tablas. Ejemplo: cliente_id en la tabla pedidos.
SQL
Lenguaje estructurado para consultar bases de datos. Se pronuncia "ese-cu-ele" o "siquel". Es el idioma universal para hablar con datos.
Query / Consulta
Pregunta que haces a una base de datos. Ejemplo: "dame todos los pedidos de Madrid del último mes".
NULL
Valor vacío o desconocido. No es cero ni espacio en blanco. Es la ausencia de información. Hay que tratarlo con cuidado en los cálculos.
Índice
Estructura que acelera las búsquedas en una tabla, como el índice de un libro. Mejora el rendimiento pero ocupa espacio.
🏢 Arquitectura de datos
Data Warehouse
Almacén centralizado de datos históricos, optimizado para análisis. Diferente de una base de datos operacional: está diseñado para preguntas de negocio, no para transacciones diarias.
Data Lake
Almacén de datos en bruto, sin estructurar, en su formato original. Más barato que un DW pero más difícil de analizar sin procesarlo antes.
Data Lakehouse
Combinación de Data Lake y Data Warehouse. Almacena datos crudos pero con capacidades de análisis estructurado. Tecnologías: Delta Lake, Apache Iceberg.
ETL
Extract, Transform, Load. El proceso de sacar datos de una fuente, transformarlos y cargarlos en otro sistema. El trabajo diario de un Data Engineer.
ELT
Como ETL pero al revés: primero cargas los datos crudos y luego los transformas. Más moderno gracias al bajo coste del almacenamiento en cloud.
Pipeline
Secuencia automatizada de pasos que mueven y transforman datos de A a B. Como una cadena de montaje para los datos.
Esquema estrella
Diseño de Data Warehouse con una tabla central de hechos rodeada de tablas de dimensiones. El modelo más común en Business Intelligence.
Tabla de hechos
Tabla central en un DW que contiene las métricas del negocio (ventas, clicks, pedidos). Tiene muchas filas y apunta a tablas de dimensiones.
Tabla de dimensiones
Tablas que describen el contexto de los hechos: ¿quién? ¿cuándo? ¿dónde? Ejemplos: dim_clientes, dim_producto, dim_tiempo.
👩‍💼 Roles y profesiones
Data Analyst
Analiza datos existentes para responder preguntas de negocio. Usa SQL, Excel y herramientas de visualización. El rol más accesible para entrar al sector.
Data Engineer
Construye y mantiene los pipelines que mueven y transforman datos. Trabaja con Python, SQL, herramientas de orquestación y plataformas cloud.
Data Scientist
Aplica estadística y machine learning para extraer insights y predecir comportamientos. Necesita Python, estadística y conocimiento del negocio.
BI Developer
Crea dashboards e informes en herramientas como Power BI o Tableau. Se enfoca en visualización y en hacer los datos comprensibles para el negocio.
Analytics Engineer
Perfil híbrido entre Data Engineer y Data Analyst. Transforma los datos crudos en modelos limpios y listos para analizar. Usa mucho dbt.
Data Steward
Responsable de la calidad y gobernanza de los datos en una organización. Define reglas, estándares y quién puede acceder a qué datos.
📈 Business Intelligence (BI)
Dashboard
Panel visual con métricas e indicadores clave del negocio, actualizado en tiempo real o periódicamente. La herramienta más usada por directivos para tomar decisiones.
KPI
Key Performance Indicator. Métrica clave para medir el rendimiento. Ejemplos: tasa de conversión, churn rate, NPS, margen bruto.
Métrica
Cualquier valor medible. Los KPIs son un subconjunto especial de métricas: las más importantes para el negocio.
DAX
Lenguaje de fórmulas de Power BI. Similar a las fórmulas de Excel pero mucho más potente. Se usa para crear medidas calculadas en los modelos de datos.
Drill-down
Navegar desde una vista resumida a un nivel más detallado. Ejemplo: clic en "Ventas Q1" → ver ventas por mes → ver ventas por semana.
Slice and dice
Filtrar y segmentar los datos desde múltiples ángulos. Ejemplo: ventas por producto, por región, por canal de venta, combinando filtros.
☁️ Cloud y herramientas modernas
BigQuery
Data Warehouse serverless de Google Cloud. Se paga por los datos consultados. Muy potente para análisis de grandes volúmenes.
Snowflake
Data Warehouse en cloud multi-proveedor (AWS, Azure, GCP). Separa almacenamiento y cómputo, lo que lo hace muy escalable.
dbt
data build tool. Herramienta open source para transformar datos dentro del Data Warehouse usando SQL. El estándar moderno para Analytics Engineering.
Apache Airflow
Orquestador de pipelines de datos. Permite programar y monitorizar flujos de trabajo complejos. El estándar del sector para Data Engineering.
Kafka
Plataforma de streaming de datos en tiempo real. Permite procesar millones de eventos por segundo. Muy usado en arquitecturas de datos modernos.
Spark
Motor de procesamiento distribuido para grandes volúmenes de datos. Mucho más rápido que MapReduce. Se programa en Python (PySpark), Scala o Java.
Parquet
Formato de archivo columnar para datos. Mucho más eficiente que CSV para análisis. Es el formato preferido en ecosistemas cloud y Spark.
🔍 Calidad y gobernanza
Data Quality
Conjunto de procesos para asegurar que los datos son correctos, completos, consistentes y actualizados. Mal dato = mala decisión.
Data Governance
Marco de políticas y procesos que define cómo se gestionan, protegen y usan los datos en una organización.
Linaje de datos
Rastro del recorrido de un dato desde su origen hasta su destino. Permite responder: ¿de dónde viene este número? ¿cómo se calculó?
Deduplicación
Proceso de eliminar registros duplicados en un dataset. Uno de los problemas más comunes en proyectos de datos reales.
SLA
Service Level Agreement. Acuerdo sobre el tiempo máximo que puede tardar un proceso o el porcentaje de disponibilidad garantizado. "Los datos deben estar listos antes de las 8:00".
🤖 Analytics avanzado
Machine Learning
Rama de la IA donde los algoritmos aprenden patrones de los datos sin ser programados explícitamente. Ejemplo: recomendaciones de Netflix.
Modelo predictivo
Algoritmo entrenado para predecir un resultado futuro basándose en datos históricos. Ejemplo: predecir si un cliente va a cancelar su suscripción.
Feature engineering
Proceso de crear o transformar variables (features) para mejorar el rendimiento de un modelo de ML. Uno de los trabajos más importantes del Data Scientist.
A/B Testing
Experimento controlado que compara dos versiones (A y B) para ver cuál funciona mejor. Muy usado en producto y marketing digital.
Sesgo (Bias)
Error sistemático en los datos o en el modelo que produce resultados incorrectos o injustos. Un modelo con sesgo puede tomar decisiones discriminatorias.
Overfitting
Cuando un modelo aprende demasiado bien los datos de entrenamiento y falla con datos nuevos. Es como memorizar las preguntas del examen en vez de entender el tema.
Feature Store
Repositorio centralizado de features reutilizables para modelos de ML. Evita que cada equipo calcule las mismas variables de manera diferente.
LLM
Large Language Model. Modelos de IA entrenados con enormes cantidades de texto. Ejemplos: GPT-4, Claude, Gemini. Están transformando el análisis de datos no estructurados.
