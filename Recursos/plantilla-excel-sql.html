Plantilla Excel → SQL

Traduce tus fórmulas favoritas de Excel a SQL. Con ejemplos reales.

Ready4Data · Si ya sabes Excel, ya tienes la mitad del camino recorrido

SUMA / CONTAR → SUM / COUNT

Excel
=SUMA(B2:B100)
=CONTAR(B2:B100)
=CONTARA(B2:B100)
SQL
SELECT SUM(importe), COUNT(*), COUNT(email) -- no nulos FROM ventas;
COUNT(*) cuenta todas las filas. COUNT(columna) cuenta solo los no nulos. Como CONTARA en Excel.

PROMEDIO / MIN / MAX → AVG / MIN / MAX

Excel
=PROMEDIO(C2:C100)
=MIN(C2:C100)
=MAX(C2:C100)
SQL
SELECT AVG(precio), MIN(precio), MAX(precio) FROM productos;

SI → CASE WHEN

Excel
=SI(B2>1000,"Premium","Normal")

=SI(B2>1000,"A",
SI(B2>500,"B","C"))
SQL
CASE WHEN ventas > 1000 THEN 'Premium' WHEN ventas > 500 THEN 'Medio' ELSE 'Básico' END AS categoria

SUMAR.SI → SUM con WHERE / GROUP BY

Excel
=SUMAR.SI(
A2:A100,"Madrid",
B2:B100
)
SQL
SELECT SUM(ventas) FROM pedidos WHERE ciudad = 'Madrid'; -- Por todas las ciudades: SELECT ciudad, SUM(ventas) FROM pedidos GROUP BY ciudad;

CONTAR.SI → COUNT con WHERE / GROUP BY

Excel
=CONTAR.SI(
A2:A100,"Activo"
)
SQL
SELECT COUNT(*) FROM clientes WHERE estado = 'Activo'; -- Todos los estados: SELECT estado, COUNT(*) AS total FROM clientes GROUP BY estado;

BUSCARV / XLOOKUP → JOIN

Excel
-- Buscar nombre del cliente =BUSCARV(
B2, -- lo que busco Clientes!A:C, -- dónde 2, -- qué columna FALSO
)
SQL
SELECT p.id, p.fecha, c.nombre, -- "columna 2" p.importe FROM pedidos p JOIN clientes c ON p.cliente_id = c.id;
El JOIN en SQL es mucho más potente que BUSCARV: no tiene límite de columnas, es más rápido y puede cruzar múltiples tablas.

ÍNDICE + COINCIDIR → JOIN con condición

Excel
=ÍNDICE(
Precios!B:B,
COINCIDIR(
A2,
Precios!A:A,
0
)
)
SQL
SELECT p.producto_id, pr.precio -- "columna B de Precios" FROM pedidos p JOIN precios pr ON p.producto_id = pr.producto_id;

Tabla Dinámica → GROUP BY

Excel — Tabla dinámica
Filas: Ciudad
Columnas: Categoría
Valores: SUMA de Ventas
Filtro: Año = 2024
SQL
SELECT ciudad, categoria, SUM(ventas) AS total FROM pedidos WHERE YEAR(fecha) = 2024 GROUP BY ciudad, categoria ORDER BY ciudad, total DESC;

CONCATENAR / CONCAT → CONCAT

Excel
=CONCATENAR(A2," ",B2)
=A2&" "&B2
=CONCAT(A2:C2)
SQL
SELECT CONCAT(nombre, ' ', apellido) AS nombre_completo FROM clientes; -- En algunos SGBD: nombre || ' ' || apellido

IZQUIERDA / DERECHA / EXTRAE → SUBSTRING

Excel
=IZQUIERDA(A2, 3)
=DERECHA(A2, 4)
=EXTRAE(A2, 3, 5)
SQL
LEFT(codigo, 3) RIGHT(codigo, 4) SUBSTRING(codigo, 3, 5) -- posición inicio, longitud

LARGO / ESPACIOS → LENGTH / TRIM

Excel
=LARGO(A2)
=ESPACIOS(A2)
=MAYUSC(A2)
=MINUSC(A2)
SQL
LENGTH(nombre) TRIM(nombre) UPPER(nombre) LOWER(nombre)

SIERROR / SI.ERROR → COALESCE / NULLIF

Excel
=SIERROR(A2/B2, 0)

=SI(A2="", "Sin dato", A2)
SQL
-- Divide, devuelve 0 si B es 0 CASE WHEN b = 0 THEN 0 ELSE a / b END -- Valor por defecto si NULL COALESCE(email, 'Sin dato')

Reglas de oro de la traducción Excel → SQL

  • Las fórmulas en columnas de Excel = SELECT en SQL
  • Los filtros de fila de Excel = WHERE en SQL
  • Las tablas dinámicas de Excel = GROUP BY en SQL
  • El BUSCARV de Excel = JOIN en SQL (mucho más potente)
  • El SI() de Excel = CASE WHEN en SQL
La diferencia clave: en Excel trabajas fila por fila. En SQL trabajas con conjuntos de datos enteros. Eso hace SQL mucho más rápido para grandes volúmenes.
