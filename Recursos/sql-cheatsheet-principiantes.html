SQL Cheatsheet
para Principiantes

Las 20 consultas que usarás el 80% del tiempo

Ready4Data · ready4data.com · Con ejemplos reales en castellano

1. SELECT — Leer datos

Seleccionar todas las columnas

SELECT * FROM clientes;

Seleccionar columnas específicas

SELECT nombre, email, ciudad
FROM clientes;
Evita el SELECT * en producción. Especifica siempre las columnas que necesitas.

2. WHERE — Filtrar filas

SELECT nombre, ventas
FROM pedidos
WHERE ventas > 1000;

-- Múltiples condiciones
SELECT * FROM clientes
WHERE ciudad = 'Madrid' AND activo = TRUE;

3. ORDER BY — Ordenar resultados

SELECT nombre, ventas
FROM pedidos
ORDER BY ventas DESC; -- DESC = mayor a menor, ASC = menor a mayor

4. LIMIT — Limitar resultados

SELECT * FROM productos
ORDER BY precio DESC
LIMIT 10; -- Solo los 10 primeros

5. COUNT, SUM, AVG, MIN, MAX — Funciones de agregación

SELECT
  COUNT(*) AS total_pedidos,
  SUM(importe) AS ventas_totales,
  AVG(importe) AS ticket_medio,
  MIN(importe) AS pedido_minimo,
  MAX(importe) AS pedido_maximo
FROM pedidos
WHERE estado = 'completado';

6. GROUP BY — Agrupar y resumir

SELECT ciudad, COUNT(*) AS num_clientes, SUM(ventas) AS total
FROM clientes
GROUP BY ciudad
ORDER BY total DESC;
Todas las columnas en el SELECT que no son funciones de agregación deben estar en el GROUP BY.

7. HAVING — Filtrar grupos

SELECT ciudad, COUNT(*) AS clientes
FROM clientes
GROUP BY ciudad
HAVING COUNT(*) > 50; -- Solo ciudades con más de 50 clientes
WHERE filtra filas. HAVING filtra grupos (después del GROUP BY).

8. JOIN — Combinar tablas

-- INNER JOIN: solo coincidencias en ambas tablas
SELECT p.id, p.fecha, c.nombre, p.importe
FROM pedidos p
INNER JOIN clientes c ON p.cliente_id = c.id;

-- LEFT JOIN: todos los de la izquierda + coincidencias de la derecha
SELECT c.nombre, p.importe
FROM clientes c
LEFT JOIN pedidos p ON c.id = p.cliente_id;
Tipo JOINQué devuelve
INNER JOINSolo filas que coinciden en ambas tablas
LEFT JOINTodas las filas de la tabla izquierda + coincidencias
RIGHT JOINTodas las filas de la tabla derecha + coincidencias
FULL JOINTodas las filas de ambas tablas

9. Alias — Renombrar columnas y tablas

SELECT
  nombre AS "Nombre del cliente",
  COUNT(*) AS num_pedidos
FROM clientes c -- alias de tabla: c
JOIN pedidos p ON c.id = p.cliente_id
GROUP BY nombre;

10. DISTINCT — Eliminar duplicados

SELECT DISTINCT ciudad FROM clientes; -- ciudades únicas

11. CASE WHEN — Condicional (el IF de SQL)

SELECT nombre,
  CASE
    WHEN ventas > 10000 THEN 'Premium'
    WHEN ventas > 5000  THEN 'Estándar'
    ELSE 'Básico'
  END AS categoria
FROM clientes;

12. Subconsultas (Subqueries)

SELECT nombre, ventas
FROM clientes
WHERE ventas > (SELECT AVG(ventas) FROM clientes);
-- Clientes con ventas por encima de la media

13. CTEs (WITH) — Consultas más legibles

WITH ventas_ciudad AS (
  SELECT ciudad, SUM(importe) AS total
  FROM pedidos p
  JOIN clientes c ON p.cliente_id = c.id
  GROUP BY ciudad
)
SELECT * FROM ventas_ciudad
WHERE total > 50000;
Los CTEs hacen el código más legible. Úsalos cuando tu consulta tiene más de 3-4 pasos.

14. Funciones de fecha

-- Año, mes, día de una fecha
SELECT YEAR(fecha), MONTH(fecha), DAY(fecha) FROM pedidos;

-- Diferencia entre fechas (días)
SELECT DATEDIFF(fecha_entrega, fecha_pedido) AS dias_entrega
FROM pedidos;

15. Funciones de texto

UPPER(nombre)        -- ANTONIO
LOWER(nombre)        -- antonio
TRIM(nombre)         -- elimina espacios al inicio/fin
CONCAT(a, ' ', b)    -- une texto
LENGTH(nombre)       -- número de caracteres
SUBSTRING(txt,1,3)   -- primeros 3 caracteres

16. NULL — Gestión de vacíos

-- Sustituir NULL por un valor
SELECT nombre, COALESCE(telefono, 'Sin teléfono') AS tel
FROM clientes;

-- Filtrar nulos
WHERE email IS NOT NULL

17. INSERT — Insertar datos

INSERT INTO clientes (nombre, email, ciudad)
VALUES ('Ana García', 'ana@email.com', 'Madrid');

18. UPDATE — Modificar datos

UPDATE clientes
SET ciudad = 'Barcelona', activo = TRUE
WHERE id = 42;
¡Siempre usa WHERE en UPDATE y DELETE! Sin WHERE modificas TODAS las filas.

19. DELETE — Eliminar datos

DELETE FROM clientes
WHERE activo = FALSE AND ultimo_pedido < '2023-01-01';

20. Orden de ejecución SQL
